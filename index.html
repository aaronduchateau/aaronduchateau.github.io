LotHopper

PHP Laravel, PostgresSQL=>PostGis, Leaflet

The true record of 'who owns what' property, is still managed at the county level. Typically this data (taxlot data) is managed offline, by GIS specialists. My online software 'LotHopper', utilizes translation layers to standardize a single interface to display taxlot data from any county, regardless of the data's schema. I then created an intuitive interface to explore and query this data. Currently you can search by many parameters, joins of those parameters, and drawn polygon. By utilizing a map server (broken out as a service from the main application) I can create rich spatial queries against county data that exceeds that of what county websites provide. As the user interacts and with the application they ultimately can gain fulfillment of their search criteria through reports, that allow print and editing capabilities.
